# Pipeline Stage Summary Through Stage 9

## One-paragraph overview
The pipeline progressed from broad hotspot-guided exploration to narrow, local-maturation design. Early stages used RFdiffusion + ProteinMPNN + RF2 to identify which hotspot logic was worth pursuing. Mid stages turned that search into concrete shortlist refinement. Late stages stopped broad redesign and instead tested whether local support around the evolving interface logic—especially CDR1-related behavior—could stabilize a viable alternative binding family.

## Stage-by-stage interpretation

### Phase 0: smoke test
- verifies the local execution chain
- confirms that phase orchestration, directory creation, and wrappers behave as expected

### Phase 1: coarse hotspot pilot
- screens broad hotspot families
- asks which antigen-side logic is worth deeper exploration
- produces the first combination-level shortlist

### Phase 2: focused pilot
- narrows Phase 1 winners into a smaller set of combinations
- shifts from “many ideas” to “plausible campaign families”

### Phase 3: main campaign
- expands the shortlisted hotspot logic at production scale
- produces the main pre-H2 candidate pool
- establishes the strong later-stage role of the `campaign_C_A_plus_pocket_rim_HBGA_adjacent_*` family

### Phase 4: H2 refinement
- optimizes the H2-focused shortlist
- creates the immediate pre-rescue / pre-local-maturation candidate base

### Phase 5: CDR1 rescue pilot
- explicitly tests whether CDR1 rescue logic can improve the evolving designed family
- ranks rescue conditions rather than committing immediately to one final line

### Phase 6: CDR1 rescue expansion
- scales the best rescue condition(s)
- produces a larger rescue-ranked candidate panel

### Phase 7 / 8 interpretation layer
- uses external AF3 analysis to compare designed candidates against WT
- makes clear that later-stage success depends less on recovering WT exactly and more on stabilizing a viable alternative interface logic
- identifies local support as a meaningful mechanistic lever

### Phase 9: local maturation branch expansion
- returns to the Test1-derived local maturation branches
- compares broader versus more compact alternative-mode behavior at larger sequence count
- produces the final candidate landscape used for downstream CA translation

## What changed scientifically over the project

### Early view
- broad hotspot exploration
- main question: “Where can binding plausibly happen?”

### Mid-project view
- surrogate filtering with RF2 became the key narrowing engine
- main question: “Which designs are structurally self-consistent enough to keep?”

### Late-project view
- AF3-based interpretation changed the optimization question
- main question became: “Can we stabilize a compact, support-coupled alternative binding mode without forcing the system back to WT?”

## Final conceptual takeaways
1. WT remains the highest-confidence stability reference.
2. The designed family does not simply converge onto WT.
3. CDR1-related support logic becomes the late-stage differentiator.
4. Compact alternative-pose designs are more informative than broad weakly spread alternatives.
